# Changelog

## @auto-skeleton/lit v0.0.5 ‚Äî May 2026 üéâ

First release of the Lit & Web Components package.

### Features

- **`<auto-skeleton>` custom element** ‚Äî zero-config skeleton loaders as a native web component, works in any framework or vanilla JS
- **Recursive Shadow DOM traversal** ‚Äî the scanner crosses Shadow DOM boundaries automatically, so deeply nested Lit components are discovered without configuration
- **`skeleton-id` attribute** ‚Äî unique cache key per instance (uses `skeleton-id` rather than `id` to avoid shadowing the native DOM attribute)
- **Property binding support** ‚Äî use `.loading=${bool}` and `.options=${{...}}` in Lit templates for ergonomic integration
- **Slotted content scanning** ‚Äî uses `slot.assignedElements()` to scan light-DOM content with correct coordinate offsets
- **All React package features** ‚Äî wave/pulse/none animations, dual-layer caching, watch mode, debug mode, CSS variable theming, reduced-motion support
- **`clearCachedBones` / `clearAllCachedBones`** ‚Äî programmatic cache control exported from the package

### Install

```bash
npm install @auto-skeleton/lit lit
```

## v0.0.3 ‚Äî May 2026

### Features

- **`AutoSkeleton` component** ‚Äî wraps any subtree, scans the live DOM on first render, and replaces it with a pixel-accurate skeleton overlay when `loading` is `true`
- **`useAutoSkeleton` hook** ‚Äî low-level hook for custom integrations, returns `ref`, `bones`, and `isScanning`
- **Data attribute overrides** ‚Äî `data-skeleton-container`, `data-skeleton-shape`, `data-skeleton-lines`, `data-skeleton-ignore`
- **Wave and pulse animations** ‚Äî controlled via `options.animation`
- **Dual-layer caching** ‚Äî bones cached in memory and `sessionStorage`, keyed by `id` + viewport width
- **Watch mode** ‚Äî `ResizeObserver`-driven re-scan when layout changes while not loading
- **Debug mode** ‚Äî `options.debug` outlines detected bones with dashed borders
- **CSS variable theming** ‚Äî `--as-base` and `--as-highlight`
- **Reduced motion support** ‚Äî animations disabled automatically when `prefers-reduced-motion: reduce` is set
- **`@auto-skeleton/core`** ‚Äî framework-agnostic DOM scanner, published separately
