# Docs Site Design

**Date:** 2026-05-12  
**Status:** Approved

---

## Goal

Build a Nextra-based documentation site for `auto-skeleton` that lives at `riazzahmedm.github.io/react-auto-skeleton/docs`. The existing demo at `/` stays untouched.

---

## Architecture

A new `/docs-site` workspace inside the existing monorepo. Nextra (Next.js) builds static HTML via `next export`, deployed to GitHub Pages under the `/docs` subpath by a dedicated GitHub Actions workflow.

```
auto-skeleton/
  docs-site/
    pages/
      index.mdx                  ← redirect to /docs/getting-started
      docs/
        getting-started.mdx
        how-it-works.mdx
        api-reference.mdx
        data-attributes.mdx
        theming.mdx
        nextjs.mdx
    theme.config.tsx              ← logo, nav, footer
    next.config.mjs               ← basePath: /react-auto-skeleton/docs
    package.json
  .github/
    workflows/
      deploy-docs.yml             ← build + export + deploy to Pages
```

## Deployment

- Workflow file: `.github/workflows/deploy-docs.yml`
- Trigger: push to `main`
- Steps: checkout → Node 20 → `npm ci` → `npm run build --workspace docs-site` → deploy `docs-site/out/` to GitHub Pages
- `next.config.mjs` sets `basePath: '/react-auto-skeleton/docs'` and `output: 'export'`
- Existing `deploy-demo.yml` is unchanged

---

## Constraints

- Static export only (`output: 'export'`) — no server-side features
- Nextra v2 (compatible with Next.js 13 static export)
- No custom domain required — GitHub Pages subpath is sufficient
- `docs-site` added to npm workspaces in root `package.json`
